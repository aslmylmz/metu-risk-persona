\section{Explosion Model}
% ──────────────────────────────────────────────────────────────────────────

The original BART by Lejuez et al.\ (2002) works by pre-drawing a single breakpoint
from a uniform distribution and popping the balloon deterministically the moment
that threshold is hit. Our version does something different. We went with a
\textbf{sequential independent Bernoulli model} instead, where on each pump
attempt $k$ the balloon has a chance of blowing up given by

\begin{equation}
\label{eq:explosion}
P(\text{explode at pump } k) = \frac{k}{\texttt{maxPumps}}\,
\end{equation}

On the client side, the check looks like this:

\begin{lstlisting}[style=tsstyle,caption={Explosion check (TypeScript, client-side).}]
const newPumps = currentBalloon.pumps + 1;
const explosionProbability = newPumps / maxPumps;
const explode = newPumps >= maxPumps
             || Math.random() < explosionProbability;
\end{lstlisting}

There are two things worth noting here.
First, because there is no hidden breakpoint drawn ahead of time, each individual
pump is a genuinely random event. The balloon will always pop once you reach
\texttt{maxPumps}, but until then every press of the button is its own coin flip.
Second, and maybe less obvious, the expected-value-optimal place to stop turns out
to be well below $\texttt{maxPumps}/2$.

\subsubsection*{Deriving the EV-Optimal Stop}

If you stop at pump $s$ and the balloon has survived all $s$ pumps, you collect
$s$ units of reward. The probability of surviving through pump $s$ under our
model is:

\begin{equation}
P(\text{survive } s \text{ pumps}) = \prod_{k=1}^{s}\left(1 - \frac{k}{N}\right)
\end{equation}

so the expected value at stopping point $s$ is:

\begin{equation}
\label{eq:ev}
\text{EV}(s, N) = s \cdot \prod_{k=1}^{s}\left(1 - \frac{k}{N}\right)
\end{equation}

We find the optimal stop $s^*$ by evaluating this for every integer $s$ from
$1$ to $N$ and picking the $s$ that gives the highest EV (see
Figure~\ref{fig:ev_curves} for the full curves). The scoring engine
does exactly this via brute-force enumeration (see \texttt{\_compute\_ev\_optimal}
in \texttt{bart.py}). A continuous approximation gives $s^* \approx \sqrt{N}$,
but the actual discrete peaks are slightly below that:

\subsubsection*{Marginal-Pump Condition}

Rather than checking every integer, we can characterize the optimum through a
ratio test. Dividing the EV at $s+1$ by the EV at $s$:

\begin{equation}
\frac{\text{EV}(s+1,\,N)}{\text{EV}(s,\,N)}
  = \frac{(s+1)\;\displaystyle\prod_{k=1}^{s+1}\!\bigl(1 - \tfrac{k}{N}\bigr)}
         {s\;\displaystyle\prod_{k=1}^{s}\!\bigl(1 - \tfrac{k}{N}\bigr)}
  = \frac{s+1}{s}\;\Bigl(1 - \frac{s+1}{N}\Bigr)
\end{equation}

An additional pump is worth taking if and only if this ratio exceeds~1, i.e.

\begin{equation}
\label{eq:marginal}
\frac{s+1}{s}\;\Bigl(1 - \frac{s+1}{N}\Bigr) > 1
\qquad\Longleftrightarrow\qquad
\text{pump again.}
\end{equation}

The first factor $(s{+}1)/s$ captures the reward gain from one more pump
(always $> 1$), while the second factor $1 - (s{+}1)/N$ captures the
additional survival cost (always $< 1$). The optimal stop $s^*$ is the
largest $s$ for which the ratio still exceeds~1; at $s^* + 1$ it drops
below~1, meaning the extra explosion risk outweighs the extra reward.

\subsubsection*{Continuous Approximation}

Setting the marginal ratio~\eqref{eq:marginal} equal to~1 and treating
$s$ as continuous:

\[
\frac{s+1}{s}\;\Bigl(1 - \frac{s+1}{N}\Bigr) = 1
\]

For large $N$ with $s \ll N$, we can write $s + 1 \approx s$ in the first
factor and expand:

\[
1 \cdot \Bigl(1 - \frac{s}{N}\Bigr) \approx 1
\quad\Longrightarrow\quad
s \approx 0
\]

This crude approximation is uninformative. A better approach: take the
logarithm of EV$(s,N)$ and differentiate. Writing
$\ln\text{EV}(s,N) = \ln s + \sum_{k=1}^{s}\ln(1 - k/N)$ and
approximating the sum by an integral:

\[
\sum_{k=1}^{s}\ln\!\Bigl(1 - \frac{k}{N}\Bigr)
\;\approx\;
\int_0^s \ln\!\Bigl(1 - \frac{x}{N}\Bigr)\,dx
\;=\;
-(N - s)\ln\!\Bigl(1 - \frac{s}{N}\Bigr) - s
\]

For $s \ll N$, using $\ln(1 - s/N) \approx -s/N - s^2/(2N^2)$, the
dominant balance gives $s^* \approx \sqrt{N}$. Checking:
$\sqrt{128} = 11.31 \approx 11$,\;
$\sqrt{32} = 5.66 \approx 5$,\;
$\sqrt{8} = 2.83 \approx 2$. The approximation is decent but consistently
overshoots by about half a pump, which is why we use the exact discrete
enumeration in practice.

\subsubsection*{Expected Session Earnings Under Optimal Play}

Given the optimal stops and the EV at each, we can compute the expected
earnings for a full 30-balloon session where a player always stops at
$s^*$ for each color. Each pump on a collected balloon is worth \$0.25,
so the per-balloon expected dollar reward is $\text{EV}(s^*,N) \times \$0.25$:

Note that this is the \emph{expected} total, not the guaranteed total.
Even an EV-optimal player will lose some balloons to explosions. For

\subsection{Session Generation}

Each session consists of exactly 30 balloons (10 of each color), presented in a
random order. The shuffling is done via Fisher--Yates so that no predictable
sequence leaks through:

\begin{lstlisting}[style=tsstyle,caption={Shuffle logic (TypeScript).}]
["ORANGE", "TEAL", "PURPLE"].forEach((type) => {
    const profile = RISK_PROFILES[type];
    for (let i = 0; i < 10; i++)
        configs.push({ id: 0, color: profile.color,
                       maxPumps: profile.maxPumps });
});
for (let i = configs.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [configs[i], configs[j]] = [configs[j], configs[i]];
}
\end{lstlisting}


% ──────────────────────────────────────────────────────────────────────────
\section{Raw Event Data Format}
% ──────────────────────────────────────────────────────────────────────────

Every user action gets logged as a timestamped JSON object on the client.
We rely on \texttt{performance.now()} for timestamps (it is monotonic and
sub-millisecond) rather than \texttt{Date.now()}, which can jump around if
the system clock is adjusted mid-session. Each event carries three fields:
\texttt{type}, \texttt{timestamp}, and \texttt{payload}.

\begin{lstlisting}[style=jsonstyle,caption={A handful of raw events from the sample session (\texttt{6fe2f79a}).}]
{"type":"pump",   "timestamp":36678.70, "payload":{"color":"purple","balloon_id":1}}
{"type":"pump",   "timestamp":37262.10, "payload":{"color":"purple","balloon_id":1}}
  ...  (4 more pumps)
{"type":"collect","timestamp":40844.90, "payload":{"color":"purple","balloon_id":1}}
{"type":"pump",   "timestamp":42964.50, "payload":{"color":"teal",  "balloon_id":2}}
{"type":"pump",   "timestamp":43418.00, "payload":{"color":"teal",  "balloon_id":2}}
{"type":"pump",   "timestamp":43836.20, "payload":{"color":"teal",  "balloon_id":2}}
{"type":"explode","timestamp":43836.30, "payload":{"color":"teal",  "balloon_id":2,
                                                    "pump_count":3}}
\end{lstlisting}

The scoring engine's first job is to chop this flat stream into per-balloon groups.
A new group begins right after every \texttt{collect} or \texttt{explode} event:

\begin{lstlisting}[style=pythonstyle,caption={Balloon segmentation (Python).}]
def _segment_balloons(events):
    balloons, current = [], []
    for event in events:
        current.append(event)
        if event.type in ("collect", "explode"):
            balloons.append(current)
            current = []
    if current:
        balloons.append(current)
    return balloons
\end{lstlisting}

Running this on our sample session produces 30 balloon segments:
10~purple, 10~teal, 10~orange, as expected.


% ──────────────────────────────────────────────────────────────────────────
\section{Handling Right-Censored Data}
% ──────────────────────────────────────────────────────────────────────────

Whenever a balloon explodes, the pump count we observe is truncated; the
participant might very well have kept going if the balloon had survived.
This is essentially a right-censoring problem, and sweeping it under the rug
biases every mean downward.

To make this concrete, consider the orange balloons in our sample session. All 10
of them blew up. Their observed counts ($1, 4, 5, 2, 4, 4, 4, 2, 2, 1$)
average out to $2.9$, but those numbers are more about where the RNG happened to
cut each trial short than about the participant's actual stopping rule. With zero
collected orange balloons, we really cannot say what the person \emph{intended}
to do.

Our solution is simple: any time we want to measure \emph{what someone chose to do},
we only look at \textbf{collected (non-exploded) balloons}. A small helper function
handles this and falls back to all balloons (with a warning flag) when there are
fewer than two collected trials for a particular color:

\begin{lstlisting}[style=pythonstyle,caption={Collected-preference helper (Python).}]
MIN_COLLECTED_FALLBACK = 2

def _prefer_collected(collected, all_data, min_count=2):
    if len(collected) >= min_count:
        return collected, False      # use collected
    return all_data, True            # fallback + warning
\end{lstlisting}

For orange, this fallback fires pretty routinely. The probability of surviving even
4 pumps on an orange balloon is only about $20.5\%$
($\frac{7}{8} \times \frac{6}{8} \times \frac{5}{8} \times \frac{4}{8} \approx 0.205$),
so in most sessions you end up with very few (or zero) collected orange balloons.


% ──────────────────────────────────────────────────────────────────────────
\section{Overall Metrics}
% ──────────────────────────────────────────────────────────────────────────

\subsection{\texttt{average\_pumps\_adjusted}}

This is the classic ``adjusted'' BART score from Lejuez's original work: the mean
number of pumps across only those balloons that the participant chose to cash in.

\begin{lstlisting}[style=pythonstyle]
non_exploded_pumps = []
for balloon_events in balloons:
    pumps = sum(1 for e in balloon_events if e.type == "pump")
    terminal = next(
        (e.type for e in reversed(balloon_events)
         if e.type in ("collect", "explode")), None)
    if terminal != "explode":
        non_exploded_pumps.append(pumps)

average_pumps_adjusted = (np.mean(non_exploded_pumps)
                          if non_exploded_pumps
                          else np.mean(all_pumps))
\end{lstlisting}

The range is $[0,\, 128]$.  In the sample session, 10~balloons were collected
with counts $\{6, 6, 5, 5, 11, 7, 3, 21, 5, 3\}$, giving
$\texttt{average\_pumps\_adjusted} = 7.2$.

An important limitation: this number treats all colors the same. Someone who
pumps 5~times on every color and someone who pumps 20~times on purple but only
once on orange could end up looking identical. If you want something that
accounts for the color differences, see \texttt{rng\_normalized\_pumps}
(\S\ref{sec:rng_norm}) or the EV-ratio score (\S\ref{sec:ev_ratio}).


\subsection{\texttt{avg\_pumps\_all\_balloons}}

The plain average over every balloon, whether it was collected or not:

\begin{equation}
    \texttt{avg\_pumps\_all\_balloons} = \frac{1}{N}\sum_{i=1}^{N} \text{pumps}_i
\end{equation}

For the sample session this comes out to $168 / 30 = 5.6$. It is handy for
direct comparisons with other BART studies that report unadjusted means, though
it does understate what the participant was trying to do since exploded balloons
carry truncated counts.


\subsection{\texttt{explosion\_rate}}

Just the fraction of balloons that popped:

\begin{equation}
    \texttt{explosion\_rate} = \frac{\texttt{total\_explosions}}{\texttt{total\_balloons}}
\end{equation}

In our sample session that is $20/30 = 0.667$. Keep in mind that two people
playing exactly the same strategy can end up with noticeably different explosion
rates on any given run, since the task is stochastic.


\subsection{\texttt{mean\_latency\_between\_pumps}}

The average number of milliseconds between consecutive pumps \emph{within} the
same balloon. We throw out cross-balloon gaps (the time someone spends deciding
whether to start a new balloon) and we also filter out any intra-balloon pause
longer than 2\,s, which we treat as an outlier.

\begin{lstlisting}[style=pythonstyle]
all_intra_latencies = []
for balloon_events in balloons:
    pump_times = [e.timestamp for e in balloon_events
                  if e.type == "pump"]
    if len(pump_times) >= 2:
        diffs = np.diff(pump_times)
        all_intra_latencies.extend(diffs.tolist())

latencies = np.array(all_intra_latencies)
latencies = latencies[latencies < 2000.0]
mean_latency = np.mean(latencies)
\end{lstlisting}

As an example, here are the inter-pump intervals for balloon~1
(purple, collected at 6~pumps):
\begin{align*}
\text{Timestamps} &= [36678.7,\; 37262.1,\; 38028.8,\; 38628.6,\; 39362.1,\; 40012.1] \\
\text{Diffs (ms)} &= [583.4,\; 766.7,\; 599.8,\; 733.5,\; 650.0]
\end{align*}
Those five intervals go into the global pool. Across all 30 balloons in the
session, the overall mean turns out to be $555.4$\,ms, which is fairly deliberate.
Fast or impulsive pumpers typically sit somewhere around ${\sim}200$\,ms.


\subsection{Session-Level Counts}

The fields \texttt{total\_balloons}, \texttt{total\_pumps},
\texttt{total\_explosions}, and \texttt{total\_collections} are
straightforward tallies. They should always obey

\begin{equation}
    \texttt{total\_explosions} + \texttt{total\_collections} = \texttt{total\_balloons}
\end{equation}

Sample session: $20 + 10 = 30$\;\checkmark.


\subsection{\texttt{money\_collected} and \texttt{money\_efficiency}}
\label{sec:money}

Each pump on a collected balloon is worth \$0.25. We compute the total
money earned and compare it to the theoretical expected earnings under
EV-optimal play:

\begin{equation}
    \texttt{money\_efficiency} = \min\!\left(1,\;
        \frac{\texttt{money\_collected}}{\sum_c n_c \cdot \text{EV}(s^*_c, N_c) \cdot 0.25}
    \right)
\end{equation}

The denominator is the theoretical expected earnings under optimal play.
For each color, the EV-optimal stop $s^*$ yields a per-balloon expected
reward (equation~\eqref{eq:ev}), which we multiply by the number of
balloons and the per-pump reward:

\begin{equation}
    \text{optimal\_earnings} = \sum_c n_c \cdot \text{EV}(s^*_c,\, N_c) \cdot \$0.25
\end{equation}

With the standard 10/10/10 configuration:
\begin{align*}
    10 \times \text{EV}(11, 128) \times 0.25 &= 10 \times 6.463 \times 0.25 = \$16.16 \\
    10 \times \text{EV}(5, 32) \times 0.25   &= 10 \times 3.038 \times 0.25 = \$7.60 \\
    10 \times \text{EV}(2, 8) \times 0.25    &= 10 \times 1.313 \times 0.25 = \$3.28 \\
    \text{Total} &= \$27.03
\end{align*}

To confirm this analytical baseline against the stochastic reality of our
explosion model, we ran a Monte Carlo simulation (100\,000 sessions of
30 balloons each, always stopping at $s^*$ per color). The simulation
script lives at \texttt{scripts/monte\_carlo\_ev.py}.

Figure~\ref{fig:ev_curves} shows the EV curves for all three colors.
Each curve rises as the reward from additional pumps accumulates, peaks
at $s^*$, and then falls off as the growing explosion probability
dominates. The peak is sharp for orange (where $P(\text{explode})$ ramps
up fast) and much flatter for purple (where participants have more room
to deviate without losing much EV).

% ──────────────────────────────────────────────────────────────────────────
\section{Behavioral Intention Metrics}
% ──────────────────────────────────────────────────────────────────────────

Everything in this section tries to get at what the participant \emph{meant}
to do, so wherever possible we restrict ourselves to collected balloons.

\subsection{\texttt{rng\_normalized\_pumps}}
\label{sec:rng_norm}

This is our main risk-taking measure. For each collected balloon we divide the
pump count by that color's capacity, then average the whole lot:

\begin{equation}
    \texttt{rng\_normalized\_pumps} = \frac{1}{|\mathcal{C}|}\sum_{b \in \mathcal{C}}
        \frac{\text{pumps}_b}{\text{maxPumps}_{\text{color}(b)}}
\end{equation}

\begin{lstlisting}[style=pythonstyle]
normalized = []
for color, pumps_list in color_pumps_behavioral.items():
    cap = COLOR_PROFILES[color]["max_pumps"]
    for p in pumps_list:
        normalized.append(p / cap)

rng_normalized_pumps = np.mean(normalized)
\end{lstlisting}

This lives on $[0, 1]$. If someone scores $0.25$, they were using roughly
a quarter of each balloon's capacity on average. The normalization gets rid of
the scale difference between colors ($128$ versus $32$ versus $8$), which makes
this metric directly comparable across risk levels, something that
\texttt{average\_pumps\_adjusted} simply cannot do.


\subsection{\texttt{impulsivity\_index}}
\label{sec:impulsivity}

A composite of three components that is always computable, even when orange
has zero collected balloons:

\begin{equation}
\boxed{
    \texttt{impulsivity\_index} = 0.40 \cdot I_{\text{timing}}
    + 0.40 \cdot I_{\text{explosions}}
    + 0.20 \cdot I_{\text{orange}}
}
\end{equation}

\subsubsection*{Timing Impulsivity (40\%)}

Fast pumping that is also rhythmically consistent points to reflexive clicking
rather than deliberation:

\begin{equation}
    I_{\text{timing}} = \min\!\left(1,\;
        \left(1 - \frac{\mu_{\text{latency}}}{800}\right) \cdot
        \left(1 + 0.5 \cdot \max(0,\, 0.5 - \text{CV}_{\text{within}})\right)
    \right)
\end{equation}

The base signal is how fast they pump ($<$ 800\,ms reference). It gets
amplified when within-balloon timing CV is low, because steady fast clicking
is more impulsive than variable fast clicking.

\subsubsection*{Excess Explosions (40\%)}

We take the explosion penalty (Section~\ref{sec:explosion_penalty}) and
scale it up by $2\times$ so that moderate over-pumping shows up clearly:

\begin{equation}
    I_{\text{explosions}} = \min(1,\; 2 \cdot \texttt{explosion\_penalty})
\end{equation}

This works across all colors and never hits a data availability issue.

\subsubsection*{Orange Signal (20\%)}

Orange gets a small weight because its explosions are mainly about
whether the participant recognized that orange is dangerous, not about
impulsivity as such. If collected orange data exists, we use
$1 - \eta_{\text{orange}}$ (over-pumping past EV-optimal). Otherwise, we
fall back to the raw orange explosion rate.

The overall range is $[0, 1]$ and the metric is always computable
regardless of how many balloons were collected.


\subsection{\texttt{patience\_index} and \texttt{patience\_index\_normalized}}

These are basically the mirror image of the impulsivity index. Here we look at
the mean pump count on collected purple balloons, the safest color, where the
smart move is to pump aggressively.

\begin{align}
    \texttt{patience\_index} &= \text{mean}(\text{collected\_purple\_pumps}) \\[2pt]
    \texttt{patience\_index\_normalized} &= \text{clip}\!\left(
        \frac{\texttt{patience\_index}}{128},\; 0,\; 1\right)
\end{align}

In the sample session we get $10.2$ (raw) and $0.080$ (normalized). The
participant was not coming close to exploiting the headroom that purple balloons
offer, though they were pumping near the EV-optimal point of~11.


\subsection{\texttt{orange\_avg\_pumps}}

This is the raw mean pump count on collected orange balloons (or all orange
balloons via fallback). It lives on $[0, 8]$. Sometimes it is easier to reason
about in plain terms: ``this person pumped an average of 4~times on the
high-risk balloons.''


% ──────────────────────────────────────────────────────────────────────────
\section{Learning \& Adaptation Metrics}
% ──────────────────────────────────────────────────────────────────────────

Both metrics below are after the same thing: \emph{did the participant get
better over the course of the session?}  They just go about answering it
in different ways.

\subsection{\texttt{learning\_rate}}

We fit a simple OLS line through pump count versus trial number for each color,
using collected balloons only. The resulting slope is then weighted by $R^2$
so that a noisy fit does not contribute as much. Crucially, the ``right''
direction depends on the color: pumping \emph{more} on purple over time is
adaptive, whereas pumping \emph{less} on orange is.

\begin{lstlisting}[style=pythonstyle,caption={Regression-based learning rate (Python).}]
for color in color_trials_all:
    collected = color_trials_collected.get(color, [])
    all_t = color_trials_all[color]
    trials = collected if len(collected) >= 2 else all_t
    if len(trials) < 2:
        continue

    xs = np.array([t[0] for t in trials])
    ys = np.array([t[1] for t in trials])
    slope, _, r_value, _, _ = stats.linregress(xs, ys)
    weighted = slope * (r_value ** 2)

    if color == "orange":
        learning_slopes.append(-weighted)
    elif color == "purple":
        learning_slopes.append(weighted)
    else:                          # teal: half-weight
        learning_slopes.append(-weighted * 0.5)

learning_rate = np.clip(np.mean(learning_slopes), -1.0, 1.0)
\end{lstlisting}

The range is $[-1, 1]$. In our sample session it comes out to $0.059$.

The obvious weakness here is sample size. With only 10 trials per color
(and frequently fewer than that once you filter down to collected ones), a single
outlier trial can throw the regression off quite a bit.


\subsection{\texttt{half\_split\_learning\_rate}}

This is a more robust alternative that avoids regression entirely. For each color
we sort the collected trials by when they occurred, split them in half, and
then look at the relative change between the two halves:

\begin{equation}
    \delta_\text{color} = \frac{\bar{x}_{\text{2nd half}} - \bar{x}_{\text{1st half}}}
                                {\bar{x}_{\text{overall}}}
\end{equation}

\begin{lstlisting}[style=pythonstyle,caption={Half-split learning rate (Python).}]
for color in color_trials_all:
    trials = collected if len(collected) >= 4 else all_trials
    if len(trials) < 4:
        continue

    sorted_t = sorted(trials, key=lambda x: x[0])
    half = len(sorted_t) // 2
    m1 = np.mean([t[1] for t in sorted_t[:half]])
    m2 = np.mean([t[1] for t in sorted_t[half:]])
    m  = np.mean([t[1] for t in sorted_t])
    if m == 0:
        continue

    delta = (m2 - m1) / m
    if color == "orange":
        scores.append(-delta)
    elif color == "purple":
        scores.append(delta)
    else:
        scores.append(-delta * 0.5)

half_split_lr = np.clip(np.mean(scores), -1.0, 1.0)
\end{lstlisting}

A value of $0.2$ here means the participant's behavior improved by roughly
$20\%$ across the session. Because we are comparing group means instead of
fitting a line through a handful of scattered points, this version is
considerably less sensitive to individual outlier trials.


% ──────────────────────────────────────────────────────────────────────────
\section{Risk Calibration Metrics}
% ──────────────────────────────────────────────────────────────────────────

\subsection{\texttt{ev\_ratio\_score}}
\label{sec:ev_ratio}

The main calibration measure. Instead of measuring linear distance from
optimal stops, we measure the actual expected value the participant would
earn at their chosen stopping point and compare it to what optimal play
would yield.

For a given color with capacity $N$, the EV at stopping point $s$ is:

\begin{equation}
    \text{EV}(s, N) = s \cdot \prod_{k=1}^{s} \left(1 - \frac{k}{N}\right)
\end{equation}

The per-color efficiency is then:

\begin{equation}
    \eta_{\text{color}} = \frac{\text{EV}(\bar{s}_{\text{participant}},\, N)}
                               {\text{EV}(s^*,\, N)}
\end{equation}

where $s^*$ is the EV-optimal stop and $\bar{s}_{\text{participant}}$ is
the mean pump count on collected balloons for that color.

\subsubsection*{EV-Weighted Averaging}

The overall score is not a simple average of per-color efficiencies. Each
color is weighted by its optimal EV value, so that high-reward colors
matter more:

\begin{equation}
    \texttt{ev\_ratio\_score} = \frac{\sum_c \eta_c \cdot \text{EV}(s^*_c, N_c)}
                                     {\sum_c \text{EV}(s^*_c, N_c)} \times 100
\end{equation}

The optimal EV values are approximately:
\begin{itemize}[nosep]
    \item \textcolor{purpleballoon}{Purple} ($N=128$): $\text{EV} \approx 6.46$, weight $\approx 60\%$
    \item \textcolor{tealballoon}{Teal} ($N=32$): $\text{EV} \approx 3.04$, weight $\approx 28\%$
    \item \textcolor{orangeballoon}{Orange} ($N=8$): $\text{EV} \approx 1.31$, weight $\approx 12\%$
\end{itemize}

The reasoning: a participant who optimizes purple well is capturing more
total reward than one who optimizes orange well, because the absolute EV
at stake on purple is roughly 5x that of orange. Equal weighting would
overvalue orange performance.

\subsubsection*{Handling 0-Collected Colors}

If a color had balloons in the session but the participant collected none
of them (all exploded), that color receives $\eta = 0$. It is not skipped.
This means someone who explodes every orange balloon gets pulled down by it,
rather than having orange silently excluded from their score.

Range: $[0, 100]$.


\subsection{\texttt{explosion\_penalty}}
\label{sec:explosion_penalty}

A complementary metric to the EV-ratio. Where EV-ratio measures
how well the participant optimizes reward, explosion penalty measures
how much they over-pump beyond what optimal play would produce.

For each color, we compute the expected explosion rate under optimal play:

\begin{equation}
    P_{\text{explode}}^{*} = 1 - \prod_{k=1}^{s^*}\left(1 - \frac{k}{N}\right)
\end{equation}

The excess is:

\begin{equation}
    \text{excess}_\text{color} = \max\!\left(0,\;
        \frac{\text{explosions}_\text{color}}{\text{balloons}_\text{color}}
        - P_{\text{explode}}^{*}\right)
\end{equation}

The overall penalty is the mean of per-color excess rates, clipped to $[0, 1]$.

This metric uses \emph{all} balloons (not just collected), which means it
never runs into the 0-collected problem. Even if every orange balloon
exploded, we can still compute the excess over what optimal play would
have caused.


\subsection{\texttt{risk\_calibration\_score}}

A single number that combines the EV-ratio with the explosion penalty:

\begin{equation}
    \texttt{risk\_calibration\_score} = \texttt{ev\_ratio\_score} \times
        (1 - \texttt{explosion\_penalty})
\end{equation}

Range: $[0, 100]$. This rewards participants who both optimize reward
\emph{and} keep their explosion rate near the expected baseline.


\subsection{\texttt{ev\_efficiency\_differentiation}}
\label{sec:ev_diff}

Instead of asking ``did you pump differently on different colors?'' this
asks ``did you achieve similar EV-efficiency across risk levels?''

\begin{equation}
    \texttt{ev\_efficiency\_differentiation} = 1 - \text{CV}(\eta_{\text{purple}},\,
        \eta_{\text{teal}},\, \eta_{\text{orange}})
\end{equation}

where $\text{CV}$ is the coefficient of variation.

As with the EV-ratio, colors where the participant collected nothing receive
$\eta = 0$. This is intentional: failing to collect anything from a risk
level is a failure to differentiate strategy for that level.

A common objection here is that risk-averse people who pump 2--3 times on
everything will naturally collect orange and therefore look differentiated.
That is true, but the EV-efficiency itself will be low for purple (they are
way under the optimal), so the differentiation score might be moderate,
but their overall calibration will be poor. These two metrics are meant to
be read together.


\subsection{\texttt{flat\_strategy\_detected}}

A boolean flag that fires when the participant appears to use an
undifferentiated pumping strategy. The check looks at the CV of raw
per-color mean pump counts and the explosion pattern. If pump counts
barely change across colors and the explosion pattern matches what you
would expect from someone ignoring the color cue, the flag goes up.

When this flag is active, calibration and differentiation scores should be
read carefully. A flat-strategy player might score decently on teal
calibration by coincidence (pumping 5 on everything happens to be near
teal's optimal), but that is not strategic behavior.


\subsection{Legacy Metrics (Descriptive)}

The following metrics are retained in the output for backward compatibility
and descriptive purposes but are not used in composite scoring.

\subsubsection*{\texttt{color\_discrimination\_index}}

Cohen's $d$ between purple and orange pump counts, normalized to $[0, 1]$:

\begin{equation}
    d = \frac{\bar{x}_{\text{purple}} - \bar{x}_{\text{orange}}}
             {s_{\text{pooled}}}
    \;,\qquad
    \texttt{color\_discrimination\_index} = \text{clip}\!\left(\frac{d}{2},\; 0,\; 1\right)
\end{equation}

Limitation: it only compares two of the three colors and does not care
whether the participant was pumping the \emph{right amount}, just whether
they were pumping \emph{differently}. Superseded by
\texttt{ev\_efficiency\_differentiation}.


\subsubsection*{\texttt{risk\_sensitivity}}

Pearson $r$ between each balloon's capacity ($128$/$32$/$8$) and the
participant's pump count.

\begin{lstlisting}[style=pythonstyle]
capacities, pumps = [], []
for color, ps in color_pumps.items():
    cap = COLOR_PROFILES[color]["max_pumps"]
    for p in ps:
        capacities.append(cap)
        pumps.append(p)

r, _ = stats.pearsonr(capacities, pumps)
\end{lstlisting}

An $r$ above $0.7$ indicates strong sensitivity to risk level;
$r \approx 0$ means a flat strategy. A low $r$ is ambiguous: the person
may genuinely not differentiate or may be choosing not to. Kept for
descriptive use.


\subsubsection*{\texttt{risk\_adjustment\_score}}

Linear distance from hardcoded optimal stops (11/5/2), scored per color
with symmetric penalty for over- and under-pumping. Range: $[0, 100]$.
Superseded by \texttt{ev\_ratio\_score}, which uses the actual EV curve
shape and EV-weighted averaging rather than a linear penalty.


% ──────────────────────────────────────────────────────────────────────────
\section{Consistency Metrics}
% ──────────────────────────────────────────────────────────────────────────

\subsection{\texttt{response\_consistency}}

This is the global coefficient of variation across every intra-pump latency
in the entire session:

\begin{equation}
    \text{CV}_\text{global} = \frac{\sigma(\text{all\_latencies})}{\mu(\text{all\_latencies})}
\end{equation}

For the sample session this is $1.204$. Honestly, on its own this number is
a bit hard to interpret, because it lumps together motor jitter within a
single balloon and strategic tempo shifts between balloons. The decomposition
below is more informative.


\subsection{\texttt{within\_balloon\_consistency}}

For each balloon that has at least 3~pumps, we compute the CV of that balloon's
inter-pump intervals (again filtering out pauses over 2\,s), and then average
those CVs across all qualifying balloons:

\begin{lstlisting}[style=pythonstyle]
within_cvs = []
for balloon_events in balloons:
    ts = [e.timestamp for e in balloon_events if e.type == "pump"]
    if len(ts) >= 3:
        diffs = np.diff(ts)
        diffs = diffs[diffs < 2000.0]
        if len(diffs) >= 2 and np.mean(diffs) > 0:
            within_cvs.append(np.std(diffs) / np.mean(diffs))

within_balloon_cv = np.mean(within_cvs) if within_cvs else 0.0
\end{lstlisting}

Low values (below about $0.2$) point to steady, rhythmic pumping; anything
above $0.6$ or so starts to suggest hesitation or attentional lapses in the
middle of a balloon.


\subsection{\texttt{between\_balloon\_consistency}}

The CV of pump counts \emph{within each color separately}, then averaged
across colors. This isolates genuine strategic inconsistency (varying your
approach on the same color from trial to trial) from appropriate adaptation
across risk levels (pumping 11 on purple and 3 on teal is correct behavior,
not inconsistency).

Uses collected-only balloons per color to avoid truncation variance; falls
back to all balloons if fewer than 3 collected are available for a color.

\begin{lstlisting}[style=pythonstyle]
per_color_cvs = []
for color in color_all_pumps:
    data = color_collected_pumps.get(color, [])
    if len(data) < 3:
        data = color_all_pumps[color]
    if len(data) < 2:
        continue
    arr = np.array(data, dtype=np.float64)
    mean_val = float(np.mean(arr))
    if mean_val > 0:
        per_color_cvs.append(float(np.std(arr) / mean_val))

between_balloon_cv = np.mean(per_color_cvs) if per_color_cvs else 0.0
\end{lstlisting}

\subsubsection*{Reading the two together}

\begin{table}[H]
\centering
\begin{tabular}{@{}lll@{}}
\toprule
\textbf{Within CV} & \textbf{Between CV} & \textbf{What it suggests} \\
\midrule
Low  & Low  & Very consistent, same strategy, steady hands \\
Low  & High & Adapts strategy across balloons, but executes each one cleanly \\
High & High & Erratic across the board \\
High & Low  & One fixed strategy, but shaky execution within each trial \\
\bottomrule
\end{tabular}
\end{table}


% ──────────────────────────────────────────────────────────────────────────
\section{Composite Metrics}
% ──────────────────────────────────────────────────────────────────────────

\subsection{\texttt{adaptive\_strategy\_score}}

A single summary number for ``behavioral intelligence'' across the whole
task. It uses conditional weighting based on how well-calibrated the
participant already is, plus a fixed 10\% weight for realized money
efficiency.

\begin{equation}
\boxed{
\texttt{adaptive\_strategy} =
w_L \!\cdot\! L
+ w_C \!\cdot\! C
+ w_D \!\cdot\! D
+ 0.10 \!\cdot\! M
}
\end{equation}

where $L$ = Learning, $C$ = Calibration, $D$ = Differentiation, $M$ = Money efficiency.

\begin{table}[H]
\centering\small
\begin{tabular}{@{}lcccc@{}}
\toprule
\textbf{Condition} & \textbf{Learning} & \textbf{Calibration} & \textbf{Differentiation} & \textbf{Money} \\
\midrule
EV-ratio $\geq 80$ & 9\% & 45\% & 36\% & 10\% \\
EV-ratio $< 80$    & 36\% & 27\% & 27\% & 10\% \\
\bottomrule
\end{tabular}
\end{table}

Components:
\begin{enumerate}[nosep]
    \item Learning: half-split learning rate, mapped from $[-1,1]$ to $[0,1]$
    \item Calibration: EV-ratio score (\S\ref{sec:ev_ratio}),
          normalized to $[0,1]$
    \item Differentiation: EV-efficiency differentiation
          (\S\ref{sec:ev_diff})
    \item Money: money efficiency
\end{enumerate}

If the participant is already well-calibrated (EV-ratio $\geq 80$),
learning gets less weight because there is less room to improve.
If calibration is poor, learning is more informative.

Range: $[0, 100]$.


% ──────────────────────────────────────────────────────────────────────────
\section{Session Validation}
% ──────────────────────────────────────────────────────────────────────────

\subsection{\texttt{session\_valid}}

Before we score anything, the data goes through five sanity checks:

\begin{enumerate}
    \item \textbf{Minimum balloons.}  Fewer than 15 means we throw the session
          out entirely; 15--29 gets a warning.
    \item \textbf{Color balance.}  Each color should have around 10~balloons;
          if any color drops below~5 we flag it.
    \item \textbf{Timestamp monotonicity.}  If any $t_i < t_{i-1}$, something
          went wrong with the logging and the session is invalid.
    \item \textbf{Session speed.}  Completing 15 or more balloons in under 30
          seconds is suspiciously fast.
    \item \textbf{Pump uniformity.}  If the standard deviation of pump counts
          is below $0.5$ across 10 or more balloons, it looks like a bot or
          an auto-clicker was involved.
\end{enumerate}

\begin{lstlisting}[style=pythonstyle,caption={Bot-detection heuristic (Python).}]
pump_counts = [sum(1 for e in b if e.type == "pump")
               for b in balloons]
if len(pump_counts) >= 10 and float(np.std(pump_counts)) < 0.5:
    warnings.append(
        "Suspicious: nearly identical pump counts "
        "across all balloons")
\end{lstlisting}

\subsection{\texttt{session\_warnings}}

This is a list that collects messages from the validation step as well as from
any data-quality fallbacks that get triggered during scoring itself.

\section{K-Means Clustering Pipeline}
\label{sec:clustering}
% ──────────────────────────────────────────────────────────────────────────

\subsection{Feature Set}

Participants with prior task experience ($n = 5$) are excluded from
clustering because familiarity with the BART paradigm substantially skews
adaptive strategy and learning metrics. The analysis runs on $N = 11$.

Feature selection was done by exhaustive search across all subsets,
filtering for combinations that include \texttt{dospert\_financial} (since
BART is a monetary risk task and the convergent validity story relies on
it), requiring max pairwise $|r| < 0.75$, and rejecting solutions with
singleton clusters.

\begin{table}[H]
\centering
\begin{tabular}{@{}llp{7cm}@{}}
\toprule
\textbf{Feature} & \textbf{Source} & \textbf{What it captures} \\
\midrule
\texttt{dospert\_financial}        & DOSPERT-30 & Self-reported financial risk tolerance \\
\texttt{bart\_rng\_normalized\_pumps} & BART    & Capacity-normalized pumping across collected balloons; how close to the limit they push \\
\texttt{bart\_impulsivity\_index}     & BART    & Composite timing + explosion + orange signal (Section~\ref{sec:impulsivity}) \\
\bottomrule
\end{tabular}
\caption{Features used in clustering ($N = 11$, prior task excluded). Max pairwise $|r| = 0.495$.}
\end{table}

Two behaviorally independent BART dimensions (risk level vs.\@ risk style)
alongside the self-report measure.


\subsection{Preprocessing}

All features are standardized to zero mean and unit variance before anything
else happens:

\begin{lstlisting}[style=pythonstyle,caption={Preprocessing (Python).}]
from sklearn.preprocessing import StandardScaler

# Drop participant_id, keep only clustering features
df_numeric = df_work[CONTINUOUS_COLS].copy()

scaler = StandardScaler()
df_scaled = pd.DataFrame(
    scaler.fit_transform(df_numeric),
    columns=df_numeric.columns,
    index=df_numeric.index,
)
\end{lstlisting}

We also do a quick correlation check and flag any pairs where $|r|$ exceeds
$0.70$, just to catch redundancy before it warps the distance metric.


\subsection{Choosing $k$: Elbow and Silhouette}

We sweep $k$ from 2 through 6 and evaluate each solution using inertia
(within-cluster sum of squares) alongside the silhouette score:

\begin{lstlisting}[style=pythonstyle,caption={$k$-selection loop (Python).}]
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score

inertias, silhouettes = [], {}
for k in range(2, 7):
    km = KMeans(n_clusters=k, n_init=20, random_state=42,
                max_iter=2000)
    labels = km.fit_predict(X)
    inertias.append(km.inertia_)
    silhouettes[k] = silhouette_score(X, labels)

best_k = max(silhouettes, key=silhouettes.get)
\end{lstlisting}

limits (high normalized pumps) with the lowest impulsivity of any group:

is only moderately above average, suggesting they may not perceive their

but BART reveals fundamentally different strategies. Self-report alone
cannot distinguish deliberate from impulsive risk-taking.
